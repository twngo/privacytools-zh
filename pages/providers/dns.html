用語說明

  • DNS-over-TLS (DoT) - 加密 DNS 的安全協定，依靠指定專用的 853 端口，某些供應商則用 443 端口，尤其當 853 常被限制的防火牆所檔。它有兩種模式：
    • 自動伺機模式：當用戶試圖透過 853 端口進行 DNS-over-TLS 連線到伺服器時，不會執行憑證查驗。如果連線失敗，它會改用未加密的 DNS。
    • 嚴格模式：用戶連接到某特定域名，執行相關的憑證查驗動作。如果查驗無法通過，則不會接受 DNS 查詢。
  • DNS-over-HTTPS (DoH) - 類似 DoT，但採用 HTTPS 取代，透過 443 端口，和一般的 HTTPS 流量無異。
  • DNSCrypt - 舊式但仍常被使用的 DNS 加密方式。

如何查驗 DNS 有加密

  • DoH / DoT
    • 利用 DNSLeakTest.com 來檢查。
    • 檢查您所使用的 DNS 服務商網站，應該可找到 "您正在使用 DNS 服務"的資訊頁。例如 AdGuard 和 Cloudflare。
    • 若使用 Firefox 信任的遞迴式解析器 (recursive resolver, TRR)，在設定檔中找到 about:networking#dns。若 TRR 在此設定下有些欄位是 "true"，即代表目前正使用 DoH。
  • dnscrypt-proxy - 參考此維基頁的說明進行檢查 dnscrypt-proxy's wiki ，看看您的 DNS 是否經過加密。
  • DNSSEC - 利用 DNSSEC Resolver Test by Matthäus Wander 檢查。
  • QNAME Minimization - 執行文字指令 dig +short txt qnamemintest.internet.nl (參考 NLnet Labs 簡報檔p)。在 Windows 10 的機器，可在 PowerShell 執行 run Resolve-DnsName -Type TXT -Name qnamemintest.internet.nl指令。 應該可看到螢幕會出現以下訊息："HOORAY - QNAME minimisation is enabled on your resolver :)!"

軟體建議與其它資訊

  • 電腦端使用 DNS 加密：
    • 新版Firefox已內建 DoH 支援並把 Cloudflare 當作預設解析器，不過使用者還是可以自行設置改成其它 DoH 解析器。 目前 Mozilla 正在研究 未來打算讓美國的使用者默認啟動 DoH。
      • "DNS over HTTPS"之功能開關可在選單 -> 偏好 (about:preferences) -> 網路設定 -> 下找到"Enable DNS over HTTPS"。將 "Use Provider" 設置為 "Custom"，再輪入 DoH 供應者的地址。
      • 熟練的使用者可以透過 about:config 來開啟，請將 network.trr.custom_uri 與 network.trr.uri 的地址設置為您的 DoH 所提供的地址，並將 network.trr.mode 設為 2。最好也能把 network.security.esni.enabled 設為 True 以便啟動加密的 SNI 讓網站支援 ESNI　更不容易被追踨。
  • 手機上使用 DNS 加密：
    • Android 9 已預設提供 DoT。
      • 推薦選擇 "Private DNS provider hostname"再輸入 DoT 提供商之地址以啟動嚴格模式（請參考上方用語說明）。
    • DNSCloak - 開源的DNSCrypt DoH 工具，在 iOS 使用由the Center for the Cultivation of Technology gemeinnuetzige GmbH開發。
    • Nebulo - Android 系統上支援 DoH / DoT 的開源應用，也支援 DNS 回應與本地的 DNS 查詢記錄。
  • 本地網路的 DNS 伺服器：
    • Stubby - Linux, macOS, Windows 皆可執行的開源軟體，可在本地網路的 DNS Privacy 根存解析器使用 DoT.
    • Unbound -可驗證、遞迴、快取的域名解析工具，可在泛網路裏執行且支援 DNS-over-TLS 功能（需 1.7.3 版以上）
      • 另見 Actually secure DNS over TLS in Unbound on ctrl.blog之說明
  • 區域網路中的 DNS 伺服器：
    • Pi-hole - 安裝在樹莓派（Raspberry Pi) 區域網路裏的 DNS 伺服器，可幫區域網路裏所有的設備擋掉廣告、追蹤和惡意域名。
    • NoTrack - 類似 Pi-hole 安裝在區域網路伺服器上的域名解析工具，可過濾廣告、追蹤和惡意域名。
